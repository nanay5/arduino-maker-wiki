## 📊 评价维度（满分 100 分）

| 维度 | 分值 | 评分标准 |
|---|---|---|
| 功能实现 | 40分 | 作品是否完成核心功能（如避障、循迹） |
| 代码质量 | 20分 | 代码规范、有注释、变量名有意义 |
| 创新改进 | 20分 | 是否超出基础要求，有自己的创意 |
| 项目展示 | 10分 | 能否清晰讲解自己的作品 |
| 团队协作 | 10分 | 小组合作情况、任务分工 |

## 📝 评价等级

| 分数 | 等级 | 说明 |
|---|---|---|
| 90-100 | A+ | 功能完整 + 有创新点 + 展示出色 |
| 80-89 | A | 功能完整，展示清晰 |
| 70-79 | B | 核心功能完成，有小问题 |
| 60-69 | C | 基本跑通，需老师帮助 |
| 低于60 | D | 未完成，需补做 |
